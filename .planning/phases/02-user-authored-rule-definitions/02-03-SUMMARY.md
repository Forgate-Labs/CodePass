## Decisions Made
- Kept raw JSON editing as a mode inside the existing authored-rule modal rather than introducing a separate editor surface or external code editor dependency.
- Treated backend service validation as authoritative for raw DSL JSON so UI and persistence paths enforce the same supported kind/schema and required fields.
- Preserved the v1 product rule that active rules are user-authored only; catalog metadata remains editor scaffolding and never becomes a built-in production rule pack.
- Added startup schema initialization for existing local SQLite databases to avoid manual database deletion or migration steps during the local self-hosted workflow.
